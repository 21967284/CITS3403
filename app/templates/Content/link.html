Explanation

The link layer is the lowest layer in the Internet Protocol Suite (IP) suite. What this means is that this layer is the layer closest to the physical hardware.

You may not have given much thought to how a computer is able to do all the things its able to do based on just electricity and voltage - but there is actually a lot that needs to happen.

In this layer, we'll explore what those things are.

As you may recall, a byte is several bits put together. And a bit is just a piece of information representing either 0 or 1, true or false, on or off.

How might we use electricity to represent this information?

Well, it's actually pretty simple. We can turn on the electricity to represent on/true/1 and we can turn off the electricity to represent off/false/0.

We put several of these on and off signals together and bam! We have a byte!

That's how I, the great mighty, [byte name] came about!

Great question!

Given how much data we have in the modern environment, there are bound to be the occasional errors that arise when encoding information. There is actually a great deal of thought that has been done to create mechanisms that ensures our messages are sent and received as we intended for them to.

One of these mechanisms is the Hamming Code! I will provide a simplfied explanation for this tutorial, but if you want to learn more, you are always welcome to visit the ever-helpful Wikipedia

In essence, hamming code creates 'codewords' by grouping together several bits of information and checking bits. As their name suggest, these checking bits serve as a way to check if all the other bits are correct. So if we want to make sure our information is correct, we just check the checking bits! If they are wrong or not quiet what we expect, then we know the information sent is not correct. Sometimes, depending on what part of the information is wrong, we can even correct the wrong bit!

Yes! Computers can deal with a lot of data!

One of the ways this layer helps the computer to process lots of data is to chunked data into Frames. It may have a fancy sounding name, but in reality, frames are just a group of data.

Along with the data that we are interested in, each frame also has extra data about the frame itself. These extra data helps the computer figure out what to make of these frames and includes important information like the destination of this frame!

Transmitting data in frames also helps us make sure our data is arriving in the correct order. For example, if the data we are about to send looks like:

001010100010

And the computer receives:

110010101000

There is no way for the receiving computer to tell that it has received the information in the correct order! But with frames, if we are sending:

Frame 1, Frame 2, Frame 3

And the receiving computer receives:

Frame 2, Frame 3, Frame 1

We can tell straight away that something somewhere has gone wrong.
